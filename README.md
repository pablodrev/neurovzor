# Neuro Viewer - Платформа для диагностики дисплазии тазобедренного сустава

Современная медицинская платформа для анализа рентгеновских снимков таза с целью диагностики дисплазии тазобедренного сустава (UYOL). Система использует искусственный интеллект для автоматической сегментации анатомических структур и вычисления клинически значимых параметров.

## Основные возможности

- **Автоматическая сегментация** костных структур таза (подвздошная кость, бедренная кость, лобково-седалищная кость)
- **Расчёт клинических параметров**: ацетабулярный угол, индексы расстояний, угол между шейкой и диафизом бедренной кости (NSA), степень непрерывности дуги Shenton
- **Визуализация диагностических линий**: линия Hilgenreiner, линия Ombredanne, линия крыши впадины
- **Определение анатомических ориентиров** на основе компьютерного зрения
- **Выявление степени поражения**: нормальное развитие, дисплазия или тяжёлая дисплазия
- **Определение локализации**: левая тазобедренный сустав, правая или двусторонняя патология
- **Образовательный режим** с подробной визуализацией всех диагностических параметров

## Архитектура

Система построена на модульной монолитной архитектуре, которая обеспечивает:

- **Легкую расширяемость**: добавление новых диагностических модулей не требует изменения ядра системы
- **Разделение ответственности**: четкое разделение между ядром, API и модулями анализа
- **Масштабируемость**: готовность к добавлению новых типов диагностик (позвоночник, коленный сустав и т.д.)

## Технологический стек

### Backend

- **FastAPI 0.104.1** - веб-фреймворк с асинхронной поддержкой
- **PyTorch** - фреймворк глубокого обучения
- **YOLOv8n-seg** - модель сегментации костных структур (3 класса)
- **NumPy, SciPy, OpenCV** - обработка изображений и математические вычисления
- **Pydantic** - валидация данных
- **pydicom** - работа с DICOM файлами
- **Python 3.11**

### Frontend

- **React 18** - библиотека для построения интерфейсов
- **Vite** - быстрый сборщик JavaScript приложений
- **Radix UI** - компоненты пользовательского интерфейса
- **SCSS** - препроцессор CSS для стилизации
- **Node.js 20**

### Инфраструктура

- **Docker** - контейнеризация сервисов
- **Docker Compose** - оркестрация контейнеров

## Установка и запуск

### Быстрый старт с Docker Compose (Рекомендуется)

**Предварительные требования:**

- Docker и Docker Compose установлены
- 4+ GB оперативной памяти

**Шаги:**

1. **Клонируйте репозиторий:**

```bash
git clone https://github.com/pablodrev/neurovzor.git
cd neurovzor
```

2. **Загрузите YOLO модель:**

```bash
mkdir -p backend/weights
# Модель автоматически загрузится при первом запуске
# Или скачайте вручную: yolov8n-seg.pt в backend/weights/
```

3. **Запустите сервисы:**

```bash
docker-compose up
```

Система будет доступна по адресам:

- **Frontend**: http://localhost:3000
- **Backend API**: http://localhost:8000
- **API документация (Swagger)**: http://localhost:8000/docs

### Установка для разработки

#### Backend

```bash
cd backend

# Создайте виртуальное окружение
python -m venv venv

# Активируйте окружение
# Windows:
venv\Scripts\activate
# macOS/Linux:
source venv/bin/activate

# Установите зависимости
pip install -r requirements.txt

# Запустите сервер
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

#### Frontend

```bash
cd frontend

# Установите зависимости
npm install

# Запустите сервер разработки
npm run dev
```

Frontend будет доступен по адресу http://localhost:5173

## Использование

### Для врачей (DoctorViewer)

1. Перейдите на главную страницу
2. Загрузите рентгеновский снимок в формате DICOM, JPG или PNG
3. Система автоматически проанализирует снимок
4. Просмотрите результаты диагностики:
   - Диагноз (норма/дисплазия)
   - Вероятность точности
   - Пораженная сторона
   - Клинические параметры

### Для студентов (StudentViewer)

1. Выберите пациента или загрузите снимок
2. Система покажет:
   - Сегментированные структуры (цветные маски)
   - Диагностические линии (Hilgenreiner, Ombredanne, крыша впадины)
   - Анатомические ориентиры (точки на костях)
   - Таблицу с расчётными параметрами
3. Переключайте слои для глубокого понимания анализа

## API Эндпоинты

### Диагностика

**POST /api/v1/hip-dysplasia/analyze**

Загружает и анализирует рентгеновский снимок.

Request:

```
multipart/form-data:
  - file: <DICOM/JPG/PNG файл>
  - patient_name: <имя пациента (опционально)>
```

Response:

```json
{
  "patient_id": "uuid",
  "diagnosis": "dysplasia",
  "confidence": 0.88,
  "affected_side": "left",
  "measurements": {
    "left": {
      "acetabular_angle": 25.5,
      "h_distance": 12.3,
      "d_distance": 8.5,
      "neck_shaft_angle": 145.2,
      "shenton_gap": 2.1
    },
    "right": {...}
  },
  "educational_data": {
    "keypoints": [...],
    "masks": [...],
    "lines": [...]
  }
}
```

**GET /api/v1/patients/{patient_id}**

Получение результатов анализа пациента.

**GET /api/v1/health**

Проверка здоровья сервиса.

## Расширение функциональности

Система разработана с учётом расширяемости. Для добавления новой диагностики:

1. **Создайте новый модуль** в `backend/app/modules/`:

```python
from app.core.base_module import BaseDiagnosticModule

class MyDiagnosticModule(BaseDiagnosticModule):
    def __init__(self):
        super().__init__(
            name="my_diagnosis",
            description="Описание",
            version="1.0.0"
        )

    async def analyze(self, image_array, metadata):
        # Ваша логика анализа
        pass
```

2. **Зарегистрируйте модуль** в `app/core/lifespan.py`

3. **Добавьте API эндпоинт** в `app/api/diagnosis.py`

## Производительность

- **Время анализа**: 1-3 секунды на GPU, 3-8 секунд на CPU
- **Размер модели**: ~15 MB (YOLOv8n-seg)
- **Требования к памяти**: 2-4 GB RAM
- **Точность сегментации**: ~92% на тестовом наборе
